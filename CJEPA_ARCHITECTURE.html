C-JEPA: Implementation Notes

Reimplementation of Causal-JEPA (ICML 2026) inside stable-worldmodel, targeting PushT.   Phase 1+2 complete arXiv 2602.11389 paper ↗

1 · What is C-JEPA?

C-JEPA is an object-centric world model that learns by predicting object slot representations in latent space — never in pixel space. It combines two ideas:

  • JEPA (Joint Embedding Predictive Architecture [1]): predict the representation of masked regions from visible context, using a separate target encoder. No decoder, no reconstruction loss.
  • Object-centric slots (Slot Attention [2]): encode each frame as N learned "slot" vectors, one per object. Slots are permutation-equivariant — there is no fixed object ordering.

The causal part of C-JEPA comes from object-level history masking: randomly selected objects are hidden across the history window, forcing the model to infer their trajectories from other objects and from the anchor at time t₀. This instills a causal, object-centric inductive bias without any explicit causal graph.

The slot encoder is fully frozen during training. Only the predictor (BidirectionalTransformer), the anchor projection (φ), and the temporal positional embeddings (e_τ) are learned.

3 · Slot Encoder — VideoSAUR

VideoSAUR [3] is a self-supervised video segmentation model that produces object-level slot representations per frame. The checkpoint used here is from HazelNam/CJEPA on HuggingFace (Phase 3 integration).

VideoSAUR forward pass (per frame)
Image (C, H, W) = (3, 224, 224) │ ▼ DINOv2 ViT-S/14 [4] Patch size 14 → 16×16 = 256 patch tokens, dim=384 │ ▼ Slot Attention (2 iterations) [2] Learnable slot initializers, competitive softmax routing │ ▼ N=4 slots (N, D=128) per frame

Slot Attention [2] is a differentiable routing mechanism: N learnable slot vectors compete to explain patch tokens via normalised dot-product attention. Each iteration refines the slots. The output is permutation-equivariant — the set of slots has no canonical ordering, which is why C-JEPA uses only temporal positional encoding (no entity-axis PE).

In our implementation, DummySlotEncoder is used as a placeholder for CPU testing. It is a single linear layer (nn.Linear(C·H·W, N·D)) that produces slots with the correct shape but no meaningful semantics. VideoSAUR integration is Phase 3.

4 · Object-Level Masking and the Identity Anchor

4.1 Why object-level masking?

Standard masked autoencoders (MAE, I-JEPA) mask patches. C-JEPA masks entire object trajectories. This forces the model to reason about object permanence: "where did that object go?" from the anchor at t₀ and context from other objects.

4.2 Masking procedure (Sec 4.2 of paper)

  1. Sample the number of masked objects: |M| ~ Uniform{0, 1, …, max_masked_slots} independently per batch item.
  2. Choose |M| slot indices without replacement via random permutation.
  3. For each masked object i and each timestep τ > t₀ in the history window, replace the slot with:
    z̃_τ^i = φ(z_{t₀}^i) + e_τ (Eq. 3 in paper)
    where φ is a learned linear projection (anchor_proj) and e_τ is a learnable temporal positional embedding. The slot at t₀ is kept as the identity anchor.
  4. All slots at future timesteps (τ ≥ T_h) are always masked using the same formula.
  5. Visible slots receive temporal PE: z_τ^i + e_τ.
Why no entity-axis positional encoding? Slots are permutation-equivariant — there is no meaningful canonical ordering. Adding entity PE would break this equivariance. Only temporal PE is used, consistent with Wu et al. 2023 (SlotFormer) [5]. Object identity during inference is maintained purely through the identity anchor φ(z_{t₀}^i).

4.3 Masking implementation

# _build_masked_tokens in cjepa.py (simplified)

t_embs = self.temporal_emb(torch.arange(T))   # (T, D) — learnable e_τ

for b in range(B):
    n_mask = randint(0, max_masked_slots + 1)
    masked_objs = randperm(N)[:n_mask]

    # history masking (τ > t₀ for chosen objects)
    for i in masked_objs:
        anchor = slots_all[b, 0, i]            # identity anchor at t₀
        for tau in range(1, T_h):
            tokens[b, tau, i] = anchor_proj(anchor) + t_embs[tau]

    # future masking (all objects, all future timesteps)
    for tau in range(T_h, T):
        for i in range(N):
            anchor = slots_all[b, 0, i]
            tokens[b, tau, i] = anchor_proj(anchor) + t_embs[tau]

# add temporal PE to visible slots
tokens[visible] = tokens[visible] + t_embs[...][visible]
# concatenate action/proprio aux tokens, flatten → (B, T*N_total, D)

5 · Predictor — BidirectionalTransformer

The predictor takes the full masked token sequence and outputs predictions at every position. Unlike LeWM's causal predictor, it uses full bidirectional attention — every token attends to every other token. This lets history slots inform future predictions and vice versa (e.g. a future action can inform history recovery).

The BidirectionalBlock reuses the Attention and FeedForward modules from lewm/module.py. The only difference from LeWM's causal Block is the causal=False argument to Attention.forward: 

class BidirectionalBlock(nn.Module):
    def forward(self, x):
        x = x + self.attn(self.norm1(x), causal=False)   # ← no causal mask
        x = x + self.mlp(self.norm2(x))
        return x

N_total per timestep = N object slots + 1 action token + 1 proprio token = 6. With T_total = T_h + T_p = 3 + 1 = 4 frames, the sequence is 4 × 6 = 24 tokens.

6 · Loss Function

C-JEPA uses a single MSE loss on all masked slot positions — both the history-masked slots (object-level masking) and the future-masked slots. There is no regularisation term (contrast with LeWM's SIGReg).

L = (1/|M_all|) · Σ_{(τ,i) ∈ M_all} ‖ f(Z̃)_τ^i − sg(z_τ^i) ‖²
  • M_all — set of all masked (timestep, slot) pairs
  • f(Z̃)_τ^i — predictor output at masked position (τ, i)
  • z_τ^i — target slot from the frozen slot encoder
  • sg(·) — stop-gradient (targets come from frozen encoder, no EMA needed)

In code (forward_train in cjepa.py): 

loss = F.mse_loss(pred_slots[is_masked], slots_all[is_masked].detach())

is_masked is a boolean tensor of shape (B, T, N) marking every (batch, timestep, slot) triple that was masked. The loss is averaged over all masked positions.

No EMA target encoder. I-JEPA and similar methods use a momentum-updated target encoder. C-JEPA skips this because VideoSAUR is already frozen — the targets are stable by construction.

7 · Inference and MPC Planning

At inference time there is no history masking. The predictor receives the full history and predicts one future step at a time. To plan over a horizon T_plan, we roll out autoregressively, sliding the history window.

7.1 Hungarian Matching

Because slot ordering is arbitrary (permutation-equivariant encoder), predicted slots and goal slots may be in different orders. We use the Hungarian algorithm (scipy.optimize.linear_sum_assignment) to find the minimum-cost assignment: 

cost = torch.cdist(pred.float(), goal.float())  # (B*S, N, N)  pairwise L2
for b in range(B * S):
    _, col_ind = linear_sum_assignment(cost[b].cpu().numpy())
    pred[b] = pred[b, col_ind]   # reorder pred to match goal

Matching is applied only during MPC inference — not during training, where both prediction targets and masked tokens share object indices by construction.

7.2 Single-step prediction

_single_step_predict builds a context of T_h visible history slots + 1 future masked slot (using identity anchor from t₀), and runs the full predictor over this T_total = T_h+1 sequence: 

anchor  = slots_hist[:, 0]                         # (BS, N, D)  — t₀
fut_tok = anchor_proj(anchor) + t_embs[T_h]        # masked future token
                                                   # shape: (BS, N, D)
# stack as (BS, T_total, N_total, D), flatten, pass to predictor
preds   = predictor(flat)                          # (BS, T_total * N_total, D)
pred_slots_future = preds[:, T_h, :N]             # extract the future frame
